# B站视频分享给好友更新日志

## V0.3.0

更新：2026年5月29日

- 分享弹窗 UI 拆分为多个组件，便于独立维护弹窗、视频预览、用户列表、搜索框等区域。
- 新增“最近聊天 / 全部好友”切换，默认继续展示最近聊天列表。
- “全部好友”支持在“我的关注 / 我的粉丝”之间切换，并从对应关系列表选择接收人。
- 支持昵称搜索、300ms 防抖和滚动分页加载；粉丝搜索会提示仅筛选已加载用户。
- 切换来源、分类或搜索词时会清空已选用户，避免发送给当前列表外的用户。

## V0.2.0

更新：2026年5月29日

- 移除网页端不稳定的视频卡片私信发送逻辑，改为直接发送稳定的文本链接私信。
- 调整发送成功提示，避免继续显示卡片发送失败相关文案。

## V0.1.11

更新：2026年5月29日

- 简化分享入口定位逻辑，直接通过 `.video-share-dropdown .dropdown-bottom > .share-btns` 获取分享方式列表。
- 移除对“嵌入代码”等固定中文文案的依赖，降低 B 站文案变化导致入口注入失败的风险。

## V0.1.10

更新：2026年5月29日

- 读取视频信息前使用封面占位块，避免空图片地址显示浏览器裂图。
- 视频封面加载失败时显示占位提示，而不是破损图片。

## V0.1.9

更新：2026年5月29日

- 文本链接兜底消息增加 UP 主信息，格式为 `分享视频：【UP主：xxx】标题` 加视频链接。

## V0.1.8

更新：2026年5月29日

- 移除视频分享计数接口调用，分享结果只反映私信是否成功发送。
- 避免因 B 站返回 `71000 重复分享` 导致文本链接已发送成功但界面仍显示失败。

## V0.1.7

更新：2026年5月29日

- 私信发送请求补齐 `Origin` 和 `msg[canal_token]`，进一步贴近 B 站 Web 私信请求形状。
- 视频卡片私信仍被接口拒绝时，自动降级为文本链接私信，避免分享完全失败。
- 发送结果会区分“视频卡片发送成功”和“已降级为文本链接发送”。

## V0.1.6

更新：2026年5月29日

- 修复发送视频卡片私信时参数不符合 Web 私信接口要求，导致 `code: -400` 请求错误的问题。
- 头像地址兼容 `pic_url` 和 `http` 协议地址，并在图片加载失败时显示默认头像。
- 发送失败提示改为去重显示，避免重复点击后堆叠相同错误。

## V0.1.5

更新：2026年5月29日

- 修复最近私信联系人只显示 UID、头像无法显示的问题。
- 最近会话接口未返回完整用户资料时，会按 UID 额外查询用户昵称和头像。

## V0.1.4

更新：2026年5月29日

- 继续缩小“B站好友”入口图标和文字尺寸，并修正默认文字颜色，使其更贴近原生分享入口。

## V0.1.3

更新：2026年5月29日

- 调整“B站好友”分享入口尺寸，使图标和文字大小与原生分享入口更一致。

## V0.1.2

更新：2026年5月29日

- 将“B站好友”入口固定注入到分享方式列表中，和动态、微信、QQ 等入口并列展示。
- 修复脚本在分享面板和底部工具栏中重复创建入口的问题。

## V0.1.1

更新：2026年5月29日

- 修复关闭分享弹窗后，透明残留层仍遮挡页面操作的问题。
- 关闭弹窗时直接移除弹窗节点，并避免加载中的异步请求在关闭后重新渲染弹窗。

## V0.1.0

更新：2026年5月29日

- 新增 Bilibili 视频播放页原分享面板入口“B站好友”。
- 支持读取最近私信联系人，单选后发送当前视频分享卡片。
- 支持发送成功后调用视频分享计数接口。
- 增加登录、CSRF、接口失败、空联系人列表等基础提示。
